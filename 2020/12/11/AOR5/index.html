斯坦纳树问题

斯坦纳树问题：给定G=(V,E), C:E→R+G=(V,E), \ C:E \rightarrow R^{+}G=(V,E), C:E→R+，其中CCC满足三角不等式。对于点集S⊆VS \subseteq VS⊆V，求GGG的连通子图G′=(S′,E′), S⊆S′G'=(S', E'), \ S \subseteq S'G′=(S′,E′), S⊆S′，使得SSS中的点彼此联通，并且选中的边权和最小。显然，最优解的G′G'G′必定是一棵树，我们记作斯坦纳树，S′∖SS' \setminus SS′∖S中的点记作斯坦纳点。

斯坦纳树问题的求解是一个NP-Hard问题。在本节中，我们重点关注该问题的近似算法，同时为了简单起见，我们仅讨论完全图上的斯坦纳树。

欧氏平面上的斯坦纳树

欧氏平面上的斯坦纳树：给定平面上的nnn个点，用总长尽量短的线段把他们连接起来。注意线段可以在平面上的任意点相交。

欧氏平面上的距离关系自然满足三角不等式。

最优解中，线段在平面上除了给定点外的交点就是斯坦纳点。

从上图看出 n=3n = 3n=3 和 n=4n = 4n=4 的情况，SSS、S1S_1S1​ 和 S2S_2S2​ 是斯坦纳点。n=3n = 3n=3 时，斯坦纳点就是三角形的费马点。

满足三角不等式的斯坦纳树：2-近似算法

容易想到，如果不考虑斯坦纳点，对图GGG关于SSS的导出子图直接求最小生成树，就是斯坦纳树问题的一个可行解。在满足三角不等式的前提下，我们可以证得这就是一个222-近似的算法，下面给出证明。

算法近似比证明：

记给定点集SSS的最小生成树解TTT，斯坦纳树问题的最优解T∗T^{*}T∗。可以将T∗T^{*}T∗中的边复制一遍，得到图2T∗2T^{*}2T∗。由于图2T∗2T^{*}2T∗中的所有点的度数都是偶数且联通，那么一定存在经过SSS中所有点的欧拉回路。

从SSS中的一点出发，找到任意一条欧拉回路LLL。易知LLL的长度就是2T∗2T^{*}2T∗的边权之和。我们这样构造一个可行解：只走SSS中的点，且每个点只走一次，如果遇到了斯坦纳点或者已经走过的点就“跳跃”到下一个点。因为我们是讨论完全图上的斯坦纳树，所以跳跃是允许的，这种跳跃又称为short-cutting。这样依次走完SSS中的点，就得到一个可行解T′T'T′。

举个例子，假定下图是我们对2T∗2T^{*}2T∗构造的欧拉回路的一部分，红色代表SSS中的点，蓝色则是斯坦纳点。由于aaa之后遇到了斯坦纳点，我们要跳跃到bbb；在走到ccc之后，由于aaa已经走过了，我们要直接跳到ddd。

在满足三角不等式的前提下，直接跳过去不会比沿着欧拉回路LLL走过去的距离长，所以有C(T′)≤2C(T∗)C(T') \leq 2 C(T^{*})C(T′)≤2C(T∗)。又因为T′T'T′也是一棵SSS上的生成树，边权之和不会比最小生成树TTT小，因此C(T)≤C(T′)C(T) \leq C(T')C(T)≤C(T′)。综上所述，有C(T)≤C(T′)≤2C(T∗)C(T) \leq C(T') \leq 2C(T^{*})C(T)≤C(T′)≤2C(T∗)。近似比上界是2。

然后证明近似比下界也是2：如下图所示，构造这样一个例子，令S={v1,v2,…,vn}S = \{v_1, v_2, \dots, v_n\}S={v1​,v2​,…,vn​}。SSS中的所有点到中心点的边权值为1，其他边权值都为2。

对这个例子使用最小生成树求解，得到的树是是不经过中心点的一条链，边权和为2(n−1)2(n-1)2(n−1)。显然，该问题的最优解是利用中心点作为斯坦纳点，边权和为nnn，从而得到近似比2n−2n\frac{2n-2}{n}n2n−2​，当n→∞n \rightarrow \inftyn→∞时，近似比趋近于2。因此近似比的下界为2。

综上所述，最小生成树是满足三角不等式的斯坦纳树问题的222-近似算法。

满足三角不等式的TSP问题

根据前面的证明，我们知道普通完全图上的旅行商问题难以得到较好的近似算法。因此，我们再给该问题加上一定的限制：在距离函数CCC满足三角不等式的前提下，旅行商问题就有较好的近似算法。接下来讨论此前提下的旅行商问题。

MST松弛改进

考虑MST松弛算法的改进空间，所有边复制一遍的操作可能与最优解之间间隙比较大，我们尝试只给生成树中的奇度数点添边。显然生成树中的奇度数点会有偶数个，存在完美匹配，即求一个T∗T^{*}T∗中奇度数点之间的最小权完美匹配M∗M^{*}M∗，得到Ge=T∗∪M∗G_{e}=T^{*}\cup M^{*}Ge​=T∗∪M∗（这里的并是边集的相加，即如果两个集合中都有这条边，那么在并中会有两条一样的边）。在GeG_{e}Ge​上取任意的欧拉回路，生成一个近似解HcH_{c}Hc​，有C(Hc)≤C(T∗)+C(M∗)C(H_{c}) \leq C(T^{*}) + C(M^{*})C(Hc​)≤C(T∗)+C(M∗)。接下来证明该算法的近似比为32\frac{3}{2}23​。

哈密尔顿路径问题

哈密尔顿路径问题指在赋权完全图上求边权和最小的哈密尔顿路，这里我们同样考虑满足三角不等式的情况。类似于旅行商问题，也可以得到类似的MST松弛算法：求GGG的最小生成树T∗T^{*}T∗，做奇度数点中除去任意两点的最小权完美匹配M∗M^{*}M∗，得到Ge=T∗∪M∗G_{e}=T^{*} \cup M^{*}Ge​=T∗∪M∗再做short-cutting。其中，去掉的这两点在GeG_{e}Ge​中度数仍为奇数，作为哈密尔顿路径的起点和终点。

类似地，把最优解中除去起点、终点外的T∗T^{*}T∗奇度数点摘出来，可以得到两组匹配，且剩余一条边（因为完美匹配是去掉了两个奇度数点的），因此仍然可以证明算法近似比为32\frac{3}{2}23​。另外，要解决去掉任意两点的最小权完美匹配的求解问题，这里可以引入两个虚拟节点a,ba,ba,b，连接到所有奇度数点并设置边权为0，则在最小权完美匹配中a，b一定会各自匹配到一个奇度数点，就相当于去掉了任意两个点后的完美匹配最优解。

固定起点的哈密尔顿路径问题

如果指定了起点SSS求边权和最小的哈密尔顿路，意味着SSS的度数有可能是偶数，那么就要尝试在图GeG_{e}Ge​中调整起点SSS的度数为奇数，我们通过给SSS加一条边来解决。在T∗T^*T∗中引入误点的概念，误点包括：

  • V∖{S}V \setminus \{ S \}V∖{S}中的奇度数点

  • SSS，若SSS是偶度数点

误点的度数是需要加边调整的。这样一来，如果SSS的度数是偶数，我们就会尝试加边改为奇度数点。可以证明误点一定是奇数个：如果SSS为奇度数点，那么奇度数点共有偶数个，去掉SSS为奇数个；如果SSS为偶度数点，那么奇度数点共有偶数个，算上SSS仍为奇数个。

对所有误点求去掉任一点的最小权完美匹配M∗M^{*}M∗（类似于前面去掉任意两点求解的算法，不多阐述），做Ge=T∗∪M∗G_{e}=T^{*}\cup M^{*}Ge​=T∗∪M∗，再做short-cutting就可以得到一个解HcH_{c}Hc​。

固定起点、终点的哈密尔顿路径问题

对于固定起点和终点的哈密尔顿路径问题，算法近似比不再是32\frac{3}{2}23​。首先介绍算法过程，仍然是MST松弛算法，同样引入误点的概念：

下面证明算法近似比上界就是53\frac{5}{3}35​：

同样分析最优解和匹配之间的关系，考虑不等式C(T∗)+C(H∗)≥3C(M∗)C(T^{*})+C(H^{*}) \geq 3C(M^{*})C(T∗)+C(H∗)≥3C(M∗)是否成立，如果成立的话，由于C(T∗)C(T^{*})C(T∗)是C(H∗)C(H^{*})C(H∗)的下界，就可以得到C(M∗)≤23C(H∗)C(M^{*}) \leq \frac{2}{3}C(H^{*})C(M∗)≤32​C(H∗)。

考虑图Ge=T∗∪H∗G_{e}=T^{*} \cup H^{*}Ge​=T∗∪H∗，由于H∗H^{*}H∗是一条哈密尔顿路径，因此只有起点和终点的度数奇偶性发生改变，图GeG_{e}Ge​中仍然有偶数个误点，T∗T^{*}T∗中除了S,TS,TS,T以外的奇度数点仍然为奇度数点。容易推出，T∗T^{*}T∗中的奇度数点在GeG_{e}Ge​中都属于误点（包括S,TS,TS,T）。

由于最优解H∗H^{*}H∗中包含一组误点的匹配，即从H∗H^{*}H∗中选出所有GeG_{e}Ge​中的误点，按序两两选出误点之间的匹配边，称这个匹配为EEE，有C(E)≥C(M∗)C(E) \geq C(M^{*})C(E)≥C(M∗)。考虑图Q=Ge∖EQ=G_{e} \setminus EQ=Ge​∖E，得到的图所有点的度数都是偶数，由于E⊆H∗E \subseteq H^{*}E⊆H∗，因此T∗⊆QT^{*} \subseteq QT∗⊆Q，QQQ是联通的。综上可以得到QQQ是欧拉图，一定存在一条欧拉回路，这条回路包含了两组误点的完美匹配，满足C(Q)≥2C(M∗)C(Q) \geq 2C(M^{*})C(Q)≥2C(M∗)，所以C(T∗)+C(H∗)≥3C(M∗)C(T^{*})+C(H^{*})\geq 3C(M^{*})C(T∗)+C(H∗)≥3C(M∗)，也就证明了近似算法的近似比上界53\frac{5}{3}35​。

所以对于固定起点和终点的哈密尔顿路径问题，以上算法近似比是53\frac{5}{3}35​。
